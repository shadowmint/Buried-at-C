Hello World

This is the most base C program; it prints a single line out to the console.

There isn't much to say about this except that it gives you the very basics of C coding:

  • Importing files is done via the "#include " directive
  • Functions are in the form [type] [name] ( [args] ) { ... }
  • Programs are started by the 'main' function.
  • Comments are contained in /* ... */ blocks.

Notice that main() is returning 0; that's a standard for 'no error'. Typically if an error occurred, the output would be 1, or variety of non-zero error codes to indicate different failure types.

The printf() function is a part of the C standard library and it generally portable across all platforms.

cmake_minimum_required(VERSION 2.8)
project(HELLO)

# Debug mode
set(CMAKE_BUILD_TYPE Debug)
if(CMAKE_COMPILER_IS_GNUCXX)
  set(CMAKE_C_FLAGS "-Wall -pedantic -Wimplicit-function-declaration -Wreturn-type -Wextra -g")
endif()

# Code type
if(CMAKE_COMPILER_IS_GNUCXX)
  set(CMAKE_C_FLAGS "${CMAKE_C_FLAGS} -std=c99")
endif()

# Sources
file(GLOB_RECURSE SOURCES src/*.c)
add_executable(hello ${SOURCES})

It probably seems like overkill to be using CMake to build this program, when on a unix system you can just as easily do it by going: cc main.c

However, cmake has a couple of perks. For example, if you're using visual studio, simply choose 'visual studio project' as the cmake build type and it'll create a visual studio project to build the application.

Furthermore, the file(GLOB_RECURSE [target] [pattern]) action is fantastically useful for being able to sort your source files into various subdirectories and not have to manage them file-by-file.

If you're using a unix based system, you should be able to build the project simply by running these commands in the hello-world directory: 

  mkdir build
  cd build
  cmake ..
  make
  ./hello

There are plenty of other build systems out there; use the one that make suits the purpose and environment you're using.
